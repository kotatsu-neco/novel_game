# TEXT_GUIDE.md

## 1. 目的

この文書は、短編ノベルゲーム『返し鈴』および同系統の短編サウンドノベルを制作する際の、本文・文書・声・演出文の書き方ガイドである。

エンジン側は禁則処理・ページ分割・文字送りを担う。  
制作側は、演出上どこで止めるか、どこで読ませるか、どこで余韻を作るかを決める。

## 2. 基本原則

### 2.1 自動分割に任せすぎない

通常本文は自動分割でよい。  
ただし、恐怖演出・重要な文書・選択直前の文は、手動でページを分けることを検討する。

### 2.2 画面はスクロールしない

1ページに入らない文章は、スクロールではなくページ送りで読ませる。  
そのため、1ページの文字量を意識して書く。

### 2.3 句読点だけの行を避ける

エンジン側で禁則処理を行うが、制作側でも読点だけで長くつながる文は避ける。

## 3. 目安

### 地の文

- 1ページ8〜10行程度
- 1文は20〜35字程度を目安
- 長い情景描写は段落で分ける
- 恐怖の瞬間は短く切ってよい

### 文書

- 覚書・手紙・札は、段落単位で読ませる
- なるべく1ページまたは2ページで収める
- 明朝体で読みにくくなるため、長文にしすぎない
- 必要なら `pages` で手動ページ分割する

### 声

- 1声は短く
- 怪異の声は短いほど効きやすい
- 声の直後に地の文で説明しすぎない

### 音・演出

- `ちりん。` のような音は単独ページでもよい
- ただし乱用しない
- 文字送り速度、沈黙、背景、音をセットで考える

## 4. 使える制御

### 手動ページ

```json
{
  "type": "document",
  "pages": [
    "『一ページ目』",
    "『二ページ目』"
  ]
}
```

### インライン改行

```text
[r]
```

`[r]` は明示改行として扱う。

### インライン改ページ

```text
[p]
```

`[p]` は手動改ページとして扱う。

### 予約

```text
[l]
```

`[l]` は将来のクリック待ち命令として予約。  
v1.4ではまだ実装しない。

## 5. 書き方の例

### 悪い例

```text
雨になりそうだから、早めに行ってきなさい。
```

端末幅によっては、句点だけが次行へ送られやすい。

### 改善例

```text
雨になりそうだから。
早めに行ってきなさい。
```

または文脈に応じて、

```text
雨になりそうだから、早めに行ってきなさい
```

とし、句点を次の表示単位へ送らない判断もあり得る。

## 6. 分担

### エンジン側

- 禁則処理
- ページ分割
- 文字送り
- 全画面タップ
- バックログ
- 保存 / 読込

### 制作側

- 文長
- 段落
- 手動改ページ
- 声の短さ
- 演出上の間
- 文書の分量

## v1.7 compiler syntax

### 効果音付き本文

```md
[text se=bell_far]
遠くで、鈴が鳴った。
```

### 手動ページ

```md
[document]
[page 1]
『一ページ目』

[page 2]
『二ページ目』
```

### 注意

- compilerは固定記法のみを扱う。
- 自由なMarkdownで「次へ進む」などと書いても変換されない。
- scene ID、next、scoreは構造情報として扱う。

## v1.8 source-level metadata

`SCENARIO_SOURCE.md` の先頭には、作品単位のmetadataを置く。

```md
# content-pack
title: 作品名
gameId: work_id
saveKey: work_id_save_v01
startScene: opening

# backgrounds
- id: black_plain
  kind: cssClass
  className: bg-black_plain
```

本文を書く前に、作品名・開始シーン・背景IDがここに揃っているか確認する。
