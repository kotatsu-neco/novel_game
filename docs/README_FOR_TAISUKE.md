## 4. 今回の設計方針

### 固定したこと

- 病院の入口は「いつもの病院」に見える。
- 中に入ってから、手続き・言葉・面会札が少しずつ異常になる。
- 病室番号は317号室。
- 317という数字自体には、呪い・誕生日・事件日などの意味を持たせない。
- 主人公が病室へ向かう理由は「誤連絡を止めるため」。
- 面会札を首から掛けた時点でTRUE END不可。
- 警備員らしき人物は、本当は「病院側のものではない」が、本文では断定しない。
- 名前が変わる、名前を呼ばれる、主人公の名前に近づく方向は中心にしない。
- 中心モチーフは、普通の面会札と裏面の不自然な朱印。

## 6. 分岐構造

SCENARIO_SOURCE.md の `[choice]` ブロックは5つです。

1. 電話をどう扱うか
2. 家を出る前に何を確認するか
3. 夜間受付でどうするか
4. 面会札をどう扱うか
5. 317号室の前でどうするか

TRUE ENDは、面会札を首から掛けていない場合のみ到達できます。  
首から掛けた状態で最終分岐から受付へ戻っても、NORMAL ENDへ進みます。

## 9. 実機確認するときの見る場所

### 何を開くか

`SCENARIO_SOURCE.md` をcompilerで `main.json` に変換したうえで、スターターエンジンで開きます。

### どこを見るか

1. 最初の電話場面が自然か
2. 病院入口が怪しすぎないか
3. 夜間受付から317号室へ向かう理由が納得できるか
4. 面会札を首から掛ける選択が危険そうに見えるか
5. 首から掛けた後にTRUEへ行けない構造が成立しているか
6. 首掛けしていないNORMALで、首の跡が出ないか
7. TRUE ENDで「何を返したか分からないが、返したことだけは分かる」後味になっているか

### 合格条件

- 主人公が病院へ向かう理由に無理がない
- 入口は普通の病院に見える
- 異常は受付後から徐々に出る
- 名前の謎に寄りすぎていない
- 面会札と朱印が中心モチーフとして機能している
- TRUE ENDが完全解決になっていない
- BAD / NORMAL / TRUE の後味が区別できる
- 首掛け済みと首掛けなしのNORMAL描写が矛盾しない
